## What survived from this prompt vs. what we adapted

| Original ask | Where it ended up |
|---|---|
| Product name **husn.io** | Kept verbatim. Domain noted; not registered yet. |
| ICP = **TPMs / TPgMs at 200-thousands-employee companies** | Refined in `knowledge.md` §5 to "1,500–8,000 employees, $200M–$2B ARR, B2B SaaS / fintech first." |
| **Layer on top of email + Outlook + Gmail + Teams + Meet + meeting notes + Slack + Jira + Confluence + SharePoint + PDFs + PPTs + Excel** | Shipped 3 of these (Jira, Slack, Google = Gmail + Drive + Docs + Sheets). Microsoft + transcript tools queued. |
| **"CC an email address that pulls into our DB"** | **Killed** per `knowledge.md` §6B — Microsoft 365 DLP blocks external auto-forward by default + GDPR third-party-subject risk + two-party-consent state risk. Replaced with OAuth-only Gmail + Microsoft Graph ingestion. |
| **"Central brain — each person knows what's relevant to their role"** | Realised as the operational graph (`persons`, `projects`, `artifacts`, `mentions`, `claims`, `findings`, `briefs`) + per-persona briefs (Step 6 AI agent). |
| **"Train an AI model on all the data"** | Re-architected to **context-injection (RAG-style)** instead of fine-tuning — model weights are frozen; full project dossier is re-built every run from the live database. Same end behavior, much cheaper and more responsive to new data. |
| **"Prompt people for next meeting / update them on what changed"** | Step 6 briefs (one per persona, evidence-linked to claim ids). Notification / propagation surface deferred — agent will eventually decide who needs to know. |
| **"Concise MD files; don't waste my reading time"** | Held to: `knowledge.md` ~250 lines, `plan.md` ~300 lines, `PROGRESS.md` ~200 lines. |
| **"Parallel agents to speed up tasks"** | Used: research synthesis in Phase 0 ran 4 agents in parallel (market, personas, business, legal). Per-task agents on demand for code review. |
| **"Auditing agent running all the time over the codebase"** | Built two: (1) project-local `.claude/settings.json` PostToolUse hook (Claude-session-scoped), (2) host-side `.claude/bin/audit-watcher.{py,sh}` daemon that polls the project tree every 4s and runs an audit on any file change. The host-side one is the always-on path; runs independent of Claude. |
| **"First step: dashboard that connects all tools"** | Step 1 = connector dashboard (live at `localhost:3000`). |
| **"Second step onwards: train the model"** | Reordered: graph + claims + drift came first (Steps 2-4), agent landed Step 6. The agent reads what those prior steps produced. |
