3. **Auth.js v5** + Resend magic links + Google SSO. WorkOS/SAML deferred until an enterprise asks.
4. **Tenant signup + onboarding wizard** (connect the four sources, name the first project, pick allowlist).
5. **Stripe Checkout + portal + webhook.** One usage-metered line item (editor seats). Webhook sets `tenants.plan`.
6. **TOS / Privacy / Subprocessor pages** with lawyer-reviewed copy. Placeholder routes already exist at `/privacy`, `/terms`, `/subprocessors`.
7. **`/admin/erase` endpoint** for GDPR Art 17.
8. **`audit_events` table** + per-action audit writes.
9. **Token encryption at rest** on `connections.tokens` (AES-GCM with `TOKEN_ENCRYPTION_KEY` from env).
10. **Per-workspace Slack install manifest pattern** (customer applies the manifest in their own workspace, per `knowledge.md` §6.A).
11. **Sentry + Better Stack** (errors + uptime + logs). Free tiers fit the first ~10 customers.
12. **CI/CD** — GitHub Actions → GHCR → ssh-deploy. The cron poll is the bridge.
13. **Secrets management** — `sops + age` with an encrypted `.env.prod` in repo; one age key per operator.

While Wave 1 Stage 2 is being built, **Google OAuth verification + CASA Tier 2** runs in the background (longest pole, 6–8 weeks). First pilot customers added as Google OAuth test users (100-cap waiver) during this window.

## Operational learnings (production, 2026-06-05 onward)

These all bit us once and are now fixed. Documented so the next operator doesn't re-debug them.

1. **Worker needed `edge` network.** Worker started on `internal` only; `internal: true` blocks egress. Every external httpx call (Groq, connector APIs, OAuth refresh) died with `Temporary failure in name resolution` — silently, since the worker caught exceptions and logged inside the container. Fix: commit `327f068`, `worker.networks: [edge, internal]`.
2. **`NEXT_PUBLIC_API_URL` is a build arg, not runtime env.** Next inlines `NEXT_PUBLIC_*` into the client bundle at build time. Runtime env never reaches client code. Fix: commit `cb5b8b4`, `Dockerfile.prod ARG NEXT_PUBLIC_API_URL` + `docker-compose.prod.yml build.args`.
3. **Microsoft Drive delta cursor stuck.** After disconnect + reconnect, the new Connection inherited `extra.drive_deltas` and stayed in delta mode returning 0 changes. Fix: `POST /api/connections/{id}/reset-sync` clears cursor keys; next backfill is a full scan.
4. **`/healthz` was GET-only; healthcheck used `wget --spider` (HEAD).** 405 → unhealthy. Fix: compose now uses `wget -qO-`.
5. **Worker healthcheck inherited from api Dockerfile.** `curl /health/lite` on port 8000 — worker doesn't run uvicorn. Fix: `healthcheck.disable: true` on worker.
6. **`redeploy` as a bash alias didn't work in cron / ssh-run commands.** Fix: real script at `/usr/local/bin/redeploy`.
7. **Groq free-tier daily token cap on `llama-3.3-70b-versatile`.** ~100K tokens/day. Cron renderer (5 personas × N projects × 30 min) burns it within hours; chat 429s the rest of the UTC day. Not yet acted on. See Wave 1 Stage 2 item 1.

## Hosting

Hetzner CX32 in Falkenstein. 4 vCPU / 8 GB / 80 GB NVMe / 20 TB traffic / Ubuntu 24.04. ~€11/mo.

Containers run via `docker compose -f docker-compose.prod.yml up -d`:

- `caddy` (reverse proxy, auto SSL) — `edge`
- `web` (Next.js SSR) — `edge + internal`
- `api` (FastAPI) — `edge + internal`
- `worker` (Arq) — `edge + internal`
- `postgres` — `internal`
- `redis` — `internal`

Memory limits: api 1G, worker 1G, web 1G, postgres 2G, redis 256M, caddy 256M.

## Decisions locked in

- Hosting: Hetzner CX32 + docker compose + Caddy.
- Pilot path for Google: test-user waiver while CASA verification runs.
- LLM for Step 5/6 v2 renderer: **Groq Llama 3.3 70B** (typewriter role; NLI verifier defends faithfulness). Anthropic Haiku for the NLI verifier (JSON mode). Anthropic Sonnet kept on the table if reject rates spike.
- Auto-deploy via cron polling `origin/main` until GitHub Actions CI/CD lands in Wave 1 Stage 2.

## Compliance posture (read `knowledge.md` §6 for the full reasoning)

- MSA must include: no training on customer data, sub-processor list (Anthropic ZDR, Groq, Hetzner, Cloudflare, Resend, Sentry, Better Stack), and the explicit "output cannot be used as input to performance / disciplinary decisions" clause that defers EU AI Act Annex III classification.
- DPA required even for US customers (their EU employees → GDPR).
- GDPR Art 17 procedure: documented `/admin/erase` endpoint (Wave 1 Stage 2).
- Slack ToS: per-workspace install, not centralised SaaS pull. Per-workspace manifest pattern shipping in Wave 1 Stage 2.
- The "CC'd shadow inbox" pattern is dead; do not resurface it in marketing.
- Deal-breaker to watch: German pilots trigger Betriebsvereinbarung (works council, 2-6 months) — politely defer for 2026.
